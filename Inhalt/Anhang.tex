\section{Anhang}
\subsection{Routing in Microfrontends}
\label{subsec:microfrontends_routing}
In diesem Abschnitt wird der exemplarische Vergleich zwischen dem URL-basierten Routing und einem zustandsbasierten Routing (SPA) dargestellt.
\begin{lstlisting}[language=Javascript, caption={Exemplarischer Vergleich zwischen URL-Routing und einer SPA}, label={lst:microfrontends_routing}]
import React, { useState } from 'react';
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';
// 1. Komponenten für das State-basierte Routing (SPA)
const RoombookApp = React.lazy(() =>
  import("roombook/RoombookApp").catch(() => {
    return {
      default: () => (
        <div className="p-8 text-center text-destructive bg-destructive/10 rounded-xl m-8">
          Fehler beim Laden von Roombook! Ist die Remote-App (Port 5174)
          gestartet?
        </div>
      ),
    };
  }),
);
const InsightsApp = React.lazy(() =>
  import("insights/InsightsApp").catch(() => {
    return {
      default: () => (
        <div className="p-8 text-center text-destructive bg-destructive/10 rounded-xl m-8">
          Fehler beim Laden von Insights! Ist die Remote-App (Port 5175)
          gestartet?
        </div>
      ),
    };
  }),
);
// 2. Komponenten für das URL-basierte Routing
const RoombookApp = () => <p>Komponente: RoombookApp (URL)</p>;
const InsightsApp = () => <p>Komponente: InsightsApp (URL)</p>;
export default function RoutingVergleichApp() {
  // State für das zustandsbasierte Umschalten
  const [activeTab, setActiveTab] = useState('A');
  return (
    <BrowserRouter>
      {/* MECHANISMUS 1: URL-BASIERTES ROUTING (Ändert die Browser-URL)*/}
      <div>
        <h3>1. URL-basiertes Routing</h3>
        <nav>
          <Link to="/roombookapp"/>
          <Link to="/insightsapp"/>
        </nav>

        <Routes>
          <Route path="/roombookapp" element={<RoombookApp />} />
          <Route path="/insightsapp" element={<InsightsApp />} />
        </Routes>
      </div>
      <hr />
      {/*MECHANISMUS 2: STATE-BASIERTES ROUTING (SPA)*/}
      <div>
        <h3>2. State-basiertes Routing</h3>
        <nav>
          <button onClick={() => setActiveTab('A')}>Zeige RoombookApp</button>
          <button onClick={() => setActiveTab('B')}>Zeige InsightsApp</button>
        </nav>
        <div>
          {activeTab === 'A' && <RoombookApp />}
          {activeTab === 'B' && <InsightsApp />}
        </div>
      </div>
    </BrowserRouter>
  );
}
\end{lstlisting}
\subsection{Simulation eines Fehlers im Microfrontend \textit{error\_mf}}

Folgend wird durch das manuelle Werfen eines Errors in Verbindung mit einem Timer ein periodischer Absturz im Microfrontend \textit{error\_mf} simuliert.
Nach 30 Sekunden löst die Komponente automatisch einen Absturz aus. Im Listing \ref{lst:error_mf} ist die Implementierung dieser Logik dargestellt.

\begin{lstlisting}[language=Javascript, caption={Periodischer Absturz eines Microfrontends}, label={lst:error_mf}]
  // Periodischer Absturz Timer
  useEffect(() => {
    if (isPaused || countdown <= 0) return;

    const timer = setInterval(() => {
      setCountdown((prev) => prev - 1);
    }, 1000);

    return () => clearInterval(timer);
  }, [countdown, isPaused]);

  // Throw rendering error to trigger ErrorBoundary
  if (countdown <= 0 || manualCrash) {
    // Reset countdown for next mount recovery
    sessionStorage.setItem("error_mf_initial_crashed", "true");
    throw new Error(
      countdown <= 0
        ? "Simulierter automatischer Absturz!"
        : "Manueller Absturz simuliert!"
    );
  }
\end{lstlisting}
\subsection{Abfangen von Fehlern in Microfrontends mittels \textit{Error Boundaries}}

Hier wird die Verwendung von Error Boundaries gezeigt, um einen Absturz der Applikation zu vermeiden.

\begin{lstlisting}[language=Javascript, caption={Nutzen von Error Boundaries}, label={lst:error_boundry_mf}]
            <ErrorBoundary name="Error MF">
                  <Suspense
                    fallback={
                      <div>
                        Lade Remote App "Error_mf"...
                      </div>
                    }
                  >
                    <Error_mfApp />
                  </Suspense>
                </ErrorBoundary>
\end{lstlisting}
\subsection{Beispielabfrage aus einem Microfrontend an GraphQL}
Folgend ist eine Fetch-Abfrage aus einem Microfrontend an die GraphQL-Schnittstelle abgebildet.
\begin{lstlisting}[language=Javascript, caption={Abfrage an GraphQL aus Microfrontend}, label={lst:graphql_request}]
const response = await fetch("http://localhost:4000/graphql", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
      },
      body: JSON.stringify({query: query})
    });
\end{lstlisting}
Mit der folgenden Query
\begin{lstlisting}[language=graphql, caption={GraphQL-Query um alle Sensoren und Pumpen zu erhalten}, label={lst:graphql_query}]
query GetSensorsAndPumps {
            ifcSensors {
              name
              tag
            }
            ifcPumps {
              name
              predefinedType
            }
          }
\end{lstlisting}
